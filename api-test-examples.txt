==============================================================================
                    AUTH SERVICE - API TEST EXAMPLES
              Base URL: http://localhost:8000/auth
==============================================================================


=========================
1. SIGNUP  (POST /auth/signup)
=========================

--- Example 1: Basic signup ---
curl -X POST http://localhost:8000/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "full_name": "John Doe",
    "email": "johndoe@example.com",
    "password": "SecurePass@123",
    "country_id": "91",
    "phone": "9876543210",
    "gender": "MALE"
  }'

--- Example 2: Female user signup ---
curl -X POST http://localhost:8000/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "full_name": "Jane Smith",
    "email": "janesmith@gmail.com",
    "password": "MyStr0ng!Pass",
    "country_id": "91",
    "phone": "8765432109",
    "gender": "FEMALE"
  }'

--- Example 3: Signup with referred_by ---
curl -X POST http://localhost:8000/auth/signup \
  -H "Content-Type: application/json" \
  -d '{
    "full_name": "Ravi Kumar",
    "email": "ravikumar@yahoo.com",
    "password": "Ravi@2026!",
    "country_id": "91",
    "phone": "7654321098",
    "gender": "MALE",
    "referred_by": "3ebdd94c-7509-408d-8ff5-6261c37999a2"
  }'

==============================================================================
                              NOTES
==============================================================================

1. Replace <YOUR_ACCESS_TOKEN> / <YOUR_JWT_TOKEN> with actual JWT tokens
   obtained after successful sign-in.

2. Replace the UUIDs (e.g., 3ebdd94c-7509-408d-8ff5-6261c37999a2) with
   actual user IDs from your database.

3. Replace OTP values (e.g., 123456) with the actual OTPs received via
   SMS or email.

4. The VERIFY_USER endpoint is currently commented out in the routes file.
   You need to uncomment it before testing.

5. UPDATE_PHONE and UPDATE_EMAIL currently have auth middleware commented
   out. In production, these should require authentication.

6. The base URL assumes the server is running on port 8000.
   Check your .env file or app.constant.ts for the actual port.

7. For Windows CMD/PowerShell, replace backslash line continuations (\)
   with backtick (`) or use a single line. Example for PowerShell:

   curl -X POST http://localhost:8000/auth/signup `
     -H "Content-Type: application/json" `
     -d '{"full_name":"John Doe","email":"johndoe@example.com","password":"SecurePass@123","country_id":"91","phone":"9876543210","gender":"MALE"}'
